## Context

当前首页采用 Stack 布局，QuickActionBar 通过 Positioned 悬浮在底部导航上方。计时器使用 `screenHeight * 0.28` 固定高度，智能预测使用 `screenHeight * 0.22`。这种百分比布局在不同屏幕比例下表现不一致，长屏幕手机上计时器内容溢出。

颜色方案上，各模块使用渐变背景和呼吸动画，视觉上较为丰富但缺乏统一性。

## Goals / Non-Goals

**Goals:**
- 实现首页模块一屏可见（宝宝信息、智能预测、计时器、快捷按钮）
- 计时器在所有屏幕尺寸下按钮不溢出
- 统一视觉风格，回归朴素
- 新增成长记录快捷入口

**Non-Goals:**
- 不改变底部导航栏结构
- 不修改统计页面或时间线页面
- 不改变活动记录的数据模型
- 不添加深色模式支持（本次仅调整浅色主题）

## Decisions

### 1. 首页布局方案

**决策**：使用 Column + SingleChildScrollView 替代 Stack + Positioned

**理由**：
- 快捷按钮从悬浮改为内嵌，不再需要 Stack 层叠
- 滚动体验更自然，最近记录可滚动查看
- 代码更简洁，易于维护

**布局结构**：
```
SingleChildScrollView
├── GuestStatusBar
├── BabyInfoCard (固定 ~56px)
├── SmartPredictionCard (固定 ~120px)
├── TimerCard (内容自适应，空闲~140px，计时~210px)
├── QuickActionBar (固定 ~80px)
└── RecentActivitiesList (剩余空间)
```

### 2. 计时器高度适配方案

**决策**：移除固定高度百分比，使用内容自适应

**理由**：
- 固定百分比无法适应动态内容（计时后多出3个按钮）
- 内容自适应可确保按钮永不溢出
- 空闲状态更紧凑，计时状态自然扩展

**实现**：
- 移除 `cardHeight = screenHeight * 0.28`
- 使用 `IntrinsicHeight` 或直接让内容撑开
- 设置最小高度 `constraints: BoxConstraints(minHeight: 140)`

### 3. 视觉风格方案

**决策**：采用方案 B - 保留品牌色但弱化

**颜色调整**：
| 元素 | 原值 | 新值 |
|------|------|------|
| 计时器背景 | 渐变 + 呼吸动画 | 纯白 + 浅灰边框 |
| 智能预测背景 | 紫粉渐变 | 纯白 + Teal 强调边框 |
| 活动类型色 | 饱和色 (#4CAF50 等) | 淡化色 (#81C784 等) |
| 标题/按钮强调 | 各模块独立色 | 统一 Teal (#009688) |

### 4. 成长记录入口方案

**决策**：新增第5个快捷按钮，点击弹出半屏 Sheet

**入口设计**：
- 图标：`Icons.trending_up` 或 `Icons.child_care`
- 标签：成长
- 位置：最右侧
- 颜色：Teal 主色

**弹窗设计**：
- 复用 `HalfScreenSheet` 组件
- 内容：身高输入框 + 体重输入框
- 校验：至少填写一项
- 提交：调用 `growthRecordProvider` 保存

### 5. 呼吸动画移除

**决策**：删除 `BreathingBackground` 组件及所有使用

**理由**：
- 视觉上过于花哨，与朴素风格冲突
- 增加渲染负担（持续动画）
- 计时状态已通过颜色和标签清晰表达

## Risks / Trade-offs

### 风险1：不同屏幕尺寸下最近记录可见性

**风险**：小屏幕手机上最近记录可能完全不可见

**缓解**：
- 设置各模块最大高度限制
- 最近记录显示"查看全部"入口
- 保持滚动能力

### 风险2：快捷按钮内嵌后操作便利性

**风险**：用户习惯悬浮按钮的常驻可见性

**缓解**：
- 快捷按钮放在计时器下方，位置仍然靠上
- 一屏内可见，无需滚动即可操作

### 风险3：颜色变更影响品牌识别

**风险**：淡化颜色可能降低活动类型的辨识度

**缓解**：
- 图标颜色仍保留活动类型特征
- 标签文字明确标注类型
- 统一的 Teal 强调色增强品牌感
