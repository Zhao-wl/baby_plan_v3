## Context

当前项目已完成时间线页面的数据层实现（timeline_provider.dart），提供按日期查询活动记录的能力。时间线页面骨架（timeline_page.dart）已存在但仅显示占位文本。本设计将实现完整的时间线页面 UI，包括日期选择器、时间轴列表和活动卡片。

**当前状态**：
- ✅ 数据层：`timeline_provider.dart` 支持按宝宝ID和日期查询活动记录
- ✅ 页面骨架：`timeline_page.dart` 已配置状态保持（AutomaticKeepAliveClientMixin）
- ✅ 颜色规范：`AppColors.eat/activity/sleep/poop` 已定义
- ✅ 半屏弹窗：`HalfScreenSheet` 组件已实现，可复用

**技术约束**：
- 使用 Flutter 3.x + Riverpod 状态管理
- 遵循 Material 3 设计规范
- 支持响应式布局（手机/平板）

## Goals / Non-Goals

**Goals:**
- 实现流畅的横向滑动日期选择器，支持一周日期展示
- 实现纵向时间轴列表，清晰展示全天活动
- 正确显示活动类型颜色标识和持续时长
- 处理跨天活动的边界情况
- 集成编辑和删除交互

**Non-Goals:**
- 不修改现有的 Provider 数据层逻辑
- 不实现新的数据存储或同步功能
- 不涉及图表或统计分析（在统计页面实现）
- 不涉及推送通知或后台任务

## Decisions

### 1. 日期选择器：PageView + 周粒度

**选择**：使用 PageView 实现横向滑动，每次滑动切换一周。

**理由**：
- 与原生日历应用交互模式一致（iOS/Android 日历均使用此模式）
- 支持快速浏览多周数据
- 性能友好，只渲染当前周和相邻周

**替代方案**：
- 连续滚动列表（Horizontal ListView）：难以对齐到周边界，体验不统一
- 日历网格（TableCalendar）：信息密度过高，不符合时间线页面的简洁需求

### 2. 时间轴：CustomPainter 绘制时间线

**选择**：使用 CustomPainter 绘制左侧时间轴连线。

**理由**：
- 精确控制连线样式（虚线/实线、颜色渐变）
- 支持动态节点大小（根据活动重要性）
- 性能优于多 Container 嵌套

**替代方案**：
- Stack + Positioned 线条：实现复杂，难以对齐

### 3. 跨天活动：拆分为两天显示

**选择**：跨天活动在开始日和结束日各显示一次，分别标注实际时间。

**理由**：
- 用户可以在两天的时间线中都看到该活动
- 避免单条记录跨度过长导致视觉混乱
- 符合睡眠追踪类应用（如 Sleep Cycle）的惯例

**示例**：
```
3月9日  23:00 开始睡眠 → 显示"睡眠中..."
3月10日 07:00 结束睡眠 → 显示"睡眠结束 8小时"
```

### 4. 卡片交互：点击编辑 + 长按删除

**选择**：点击卡片打开编辑弹窗，长按显示删除确认。

**理由**：
- 与首页最近活动列表的交互模式保持一致
- 避免误操作（删除需要确认）
- 符合移动端用户习惯

### 5. 空状态：插图 + 引导文案

**选择**：当某天无记录时显示插图和引导添加的文案。

**理由**：
- 提升用户体验，避免空白页面的困惑感
- 引导用户了解如何添加记录

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大量记录时列表性能下降 | 中 | 使用 ListView.builder 实现虚拟列表，只渲染可见项 |
| 跨天活动逻辑复杂 | 低 | 在 Provider 层预处理数据，UI 层只负责展示 |
| 日期切换动画卡顿 | 低 | 使用 PageView 的预加载机制，数据使用 FutureBuilder 异步加载 |

**Trade-offs：**
- 拆分跨天活动 vs 单条显示：牺牲数据完整性换取视觉清晰度
- CustomPainter vs 预制组件：增加实现复杂度换取更好的视觉控制

## Migration Plan

本项目为新增 UI 功能，不涉及数据迁移。部署策略：

1. **开发阶段**：直接修改 `timeline_page.dart`，保持页面骨架的 IndexedStack 集成
2. **测试验证**：
   - 验证不同日期切换流畅度
   - 验证跨天活动显示正确性
   - 验证编辑/删除交互
3. **发布**：随常规版本发布，无需特殊说明
