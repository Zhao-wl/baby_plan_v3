## Context

本设计实现一个智能预测引擎，用于预测宝宝的下一项活动（吃、睡、排泄）。系统需要：

- **历史数据分析**：分析宝宝最近7天的活动记录，提取个性化模式
- **月龄基准参考**：参考按周龄划分的活动模式基准数据（0-52周）
- **加权融合**：历史数据70%权重 + 月龄基准30%权重
- **预测合并**：15分钟内的多个预测合并显示

## Goals / Non-Goals

**Goals:**
- 实现基于历史数据的活动预测算法
- 引入月龄活动模式基准数据（按周划分）
- 预测结果在首页卡片中动态展示
- 支持"打勾"操作标记已处理的预测
- 每次活动记录变更自动重新计算

**Non-Goals:**
- 不实现云端同龄宝宝数据对比
- 不实现机器学习模型（使用简单统计算法）
- 不实现预测历史记录
- 不实现多宝宝同时预测（仅当前选中宝宝）

## Decisions

### 1. 数据粒度：按周划分而非按月

**决策**：月龄基准数据按周（0-52周）划分，而非按月。

**理由**：
- 婴儿发育在早期变化快，按月粒度太粗糙
- 7天历史数据窗口与周粒度天然吻合
- 52周覆盖一整年，满足主要预测需求

**备选方案**：按月划分
- 优点：数据量更小
- 缺点：0-3个月变化快，按月粒度不够精细

### 2. 预测算法：简单加权平均

**决策**：使用加权平均算法，历史数据70% + 月龄基准30%。

**理由**：
- 简单直观，易于理解和调试
- 个性化权重更高，尊重宝宝的个体差异
- 月龄基准作为兜底，避免极端预测

**算法流程**：
```
预测时间 = 历史预测 × 0.7 + 基准预测 × 0.3
```

**备选方案**：机器学习模型
- 优点：可能更准确
- 缺点：复杂度高，需要大量数据训练，本地计算资源有限

### 3. 预测触发：实时计算

**决策**：每次活动记录变更都重新计算预测。

**理由**：
- 利用现有的 `activityDataChangeProvider` 机制
- 预测计算量小，实时计算无性能问题
- 保证预测始终基于最新数据

**备选方案**：定时刷新
- 优点：减少计算次数
- 缺点：预测可能过时，用户体验差

### 4. 数据存储：预置 JSON + SQLite 缓存

**决策**：月龄基准数据通过 JSON 预置，首次启动时加载到 SQLite。

**理由**：
- 与疫苗库数据加载模式一致
- 支持离线使用
- 便于数据更新（更新 JSON 文件即可）

**数据格式**：
```json
{
  "version": 1,
  "patterns": [
    {
      "week": 0,
      "activityType": 0,
      "intervalMinutes": 150,
      "durationMinutes": 25,
      "countPerDay": 10
    }
  ]
}
```

### 5. 已处理预测：内存标记

**决策**：已处理的预测通过内存中的 Set 记录，不持久化。

**理由**：
- 预测是实时计算的，每次重新计算都会生成新预测
- "打勾"操作仅影响当前预测的生命周期
- 重启应用后重新计算是合理行为

**备选方案**：持久化到数据库
- 优点：重启后仍有效
- 缺点：增加复杂度，且预测会自然更新

## Risks / Trade-offs

### Risk 1: 历史数据不足导致预测不准
- **风险**：新用户或数据量少时，预测可能不准确
- **缓解**：
  - 数据不足时完全依赖月龄基准（100%权重）
  - 显示"根据月龄推荐"提示，管理用户期望

### Risk 2: 进行中活动影响预测
- **风险**：宝宝正在睡觉时，不应该预测睡眠
- **缓解**：检测进行中活动，排除该类型的预测

### Risk 3: 预测时间已过
- **风险**：预测时间已过但宝宝还没有活动
- **缓解**：预测卡片显示"预测时间已过"状态，或自动重新计算

### Risk 4: 夜间频繁预测打扰用户
- **风险**：夜间22:00-06:00期间频繁推送预测
- **缓解**：夜间模式，不显示预测卡片或显示"宝宝安睡中"

## Data Model

### AgeActivityPatterns 表

```dart
class AgeActivityPatterns extends Table {
  IntColumn get id => integer().autoIncrement()();
  IntColumn get week => integer()();           // 周龄 (0-52)
  IntColumn get activityType => integer()();   // 0=吃, 2=睡, 3=排泄
  IntColumn get intervalMinutes => integer()(); // 平均间隔（分钟）
  IntColumn get durationMinutes => integer().nullable()(); // 平均持续时间
  IntColumn get countPerDay => integer()();    // 每日平均次数
  TextColumn get notes => text().nullable()(); // 备注
  IntColumn get dataVersion => integer().withDefault(const Constant(1))();
}
```

### PredictionResult 模型

```dart
class PredictionResult {
  final PredictionType type;           // 预测类型
  final DateTime predictedTime;        // 预测时间
  final double confidence;             // 置信度 (0.0-1.0)
  final String description;            // 描述
  final String? tips;                  // 建议提示
  final List<PredictionResult>? related; // 关联预测（15分钟内）
}

enum PredictionType { eat, sleep, poop }
```
